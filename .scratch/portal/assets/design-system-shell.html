04 App shell & routing — spec
FrameworkStatic / prerendered SvelteKit PWA; all math client-side; no server backend (ADR-0001, ADR-0004).
Routes/ home quick-calc · /antennas/[model] Designers · /tools/[tool] Engineering Tools · /reference/* tables.
ChromeSticky masthead: wordmark, nav (Antennas ▾ / Tools ▾ menus generated from the 24-calculator registry), the Center-Frequency input, theme toggle. Footer: Region selector + about.
StatePer-calculator UI-state ⇄ versioned readable URL params (?v=1&f=…) via the codec; Saved Projects to localStorage. Center Frequency flows from home into any calculator via the URL handoff.
ThemeToken-level light/dark; prefers-color-scheme + a toggle stamping data-theme. Self-hosted WOFF2 subsets (CSP blocks CDNs).
Quality floorResponsive to mobile, visible focus, reduced-motion respected, labelled controls — unannounced on every page.

Every calculator = shell + this component set + one compute… seam. The Designers (Antenna Models) add the geometry diagram; the Tools add their readout. Nothing here is re-decided per calculator.
